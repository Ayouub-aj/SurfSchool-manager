Project context
You work for PixelCraft Agency, an agency that transforms local businesses through digital. Your client, Taghazout Surf Expo, is a victim of its success: paper registrations and Excel files no longer make it possible to effectively track the hundreds of students who come to surf every week.

The problem : The instructors do not know the level of the students before the session, and the managers have difficulty monitoring who paid for their lessons.

As a Backend Developer, your role is:

Implement architecture MVC (Model-View-Controller) : Physically separate data logic, business logic and display for maintainable code.
Adopt it OOP (Object Oriented Programming) : Create classes (eg: Student, Lesson, Database) with strict use of encapsulation (private properties and public methods).
Manage Advanced Relationships: Design a standardized database (3 tables: users, students, lessons) with a linking table or foreign keys for registrations.
Guarantee Security & Integrity: Use PDO (Prepared Statements), password hashing and strict form validation.
User Stories (Business Needs)

HAS. The Manager (Administrator Access)
US1 (Global Management): As Manager, I want to log in to access the dashboard displaying all students and all planned courses.
US2 (Scheduling Management): As Manager, I want to create a surf session (Title, Coach, Date/Time) and register students according to their level.
US3 (Educational Monitoring): As a Manager, I want to be able to modify the level of a student (Beginner, Intermediate, Advanced) to ensure their follow-up after each lesson.
B. The Surfer (Customer Access)
US4 (Self-Registration): As a new surfer, I want to create my profile (Name, Country, Self-Assessed Level) to be listed in the school system.
US5 (My Calendar): As a connected surfer, I only want to view the list of my upcoming classes and check if my payment status is up to date ("Paid" or "Pending").
++ Bonus

MVC Router : Set up a single entry point (index.php) that directs requests to the correct controllers.

Stats Software : Display the average course occupancy rate on the dashboard (eg: "Average of 5 students per session").
